# Copyright Safety Guardrails

Use these guardrails whenever this skill references IEEE 1016-2009.

## Mandatory Rules

1. Do not copy or quote IEEE standard text, figures, or tables.
2. Use only original wording in generated SDDs and gap reports.
3. Keep references conceptual and structural (for example, section alignment), not normative text reproduction.
4. If a user requests exact IEEE wording, decline and direct them to a licensed copy of the standard.
5. Keep outputs implementation-focused and project-specific instead of reproducing generic standards language.
6. If a disclaimer is needed, include a single concise line once per deliverable (for example, "IEEE 1016-inspired internal guidance, unofficial."). Do not repeat disclaimer text in titles or every section.
7. Avoid "compliant/certified/conformant" claims; describe checks as informational internal guidance.

## Allowed Patterns

- "This section aligns with IEEE 1016 Clause 4 core content themes."
- "The document includes concern-driven viewpoint selection inspired by IEEE 1016 concepts."
- "IEEE 1016-inspired internal guidance, unofficial."

## Disallowed Patterns

- Verbatim clause content from IEEE standards.
- Copy-paste of IEEE tables or figures.
- Large near-verbatim rewrites that closely track proprietary standard wording.
- Repetitive disclaimer blocks in headings, section titles, or every section body.
